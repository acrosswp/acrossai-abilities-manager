## Phase 1 ŌĆö Design

### FIX-1 Design: Hook Migration

**Delete from `AcrossAI_Ability_Logger.php`**: entire `boot()` method (lines 86ŌĆō122) and its PHPDoc

**Delete from `Main.php`**: `$this->loader->add_action( 'plugins_loaded', $logger, 'boot', 20 );`

**Add to `Main.php` `define_public_hooks()` immediately after the `$logger` declaration**:

```php
$this->loader->add_filter( 'mcp_adapter_pre_tool_call',      $logger, 'capture_mcp_server_id',     5,      4 );
$this->loader->add_action( 'wp_before_execute_ability',      $logger, 'start_pending_entry',      10,      2 );
$this->loader->add_action( 'wp_after_execute_ability',       $logger, 'finish_pending_entry',     10,      3 );
$this->loader->add_filter( 'wp_register_ability_args',       $logger, 'wrap_permission_callback', 100001,  2 );
$this->loader->add_action( 'acrossai_ability_logger_cleanup', $logger, 'cleanup_old_logs',         10,      0 );
$this->loader->add_action( 'plugins_loaded',                 $logger, 'schedule_cleanup',          20,      0 );
```

**Why 6 calls (not 5)**: `boot()` called `$this->schedule_cleanup()` directly (not via hook). This is converted to a Loader-registered `plugins_loaded P20` call to maintain identical timing semantics.

**`$accepted_args` rationale**:
| Callback | Count | Reason |
|---|---|---|
| `capture_mcp_server_id` | 4 | Hook passes `$args, $tool_name, $mcp_tool, $server` |
| `start_pending_entry` | 2 | Hook passes `$ability_slug, $args` |
| `finish_pending_entry` | 3 | Hook passes `$ability_slug, $args, $result` |
| `wrap_permission_callback` | 2 | Filter passes `$args, $ability_slug` |
| `cleanup_old_logs` | 0 | Action scheduler; no args passed |
| `schedule_cleanup` | 0 | `plugins_loaded`; no args |

### FIX-5 Design: Source Detector Singleton

`AcrossAI_Logger_Source_Detector` holds mutable request-scoped state (`$is_mcp_context`, `$mcp_server_id`). The `DEC-UTILITY-STATIC-ONLY` decision's own criterion ("no mutable state") does not apply. Module Contract governs.

**Transformations**:
1. `git mv includes/Modules/Logger/AcrossAI_Logger_Source_Detector.php includes/Utilities/`
2. Change `namespace AcrossAI_Abilities_Manager\Includes\Modules\Logger` ŌåÆ `AcrossAI_Abilities_Manager\Includes\Utilities`
3. Add singleton scaffolding:
   ```php
   protected static $_instance = null;
   public static function instance(): self {
       if ( null === self::$_instance ) { self::$_instance = new self(); }
       return self::$_instance;
   }
   private function __construct() {}
   ```
4. Convert `private static $is_mcp_context` ŌåÆ `private $is_mcp_context = false;`
5. Convert `private static $mcp_server_id` ŌåÆ `private $mcp_server_id = null;`
6. Remove `static` keyword from all 10 public methods: `detect_source`, `is_mcp_context`, `is_rest_context`, `is_cli_context`, `is_cron_context`, `is_ajax_context`, `detect_mcp_server_id`, `set_mcp_context`, `clear_mcp_context`, `is_valid_source`
7. Update all internal `self::$is_mcp_context` / `self::$mcp_server_id` references to `$this->is_mcp_context` / `$this->mcp_server_id`
8. Update `use` import in `AcrossAI_Ability_Logger.php` to `AcrossAI_Abilities_Manager\Includes\Utilities\AcrossAI_Logger_Source_Detector`
9. Update 6 call sites: `AcrossAI_Logger_Source_Detector::method()` ŌåÆ `AcrossAI_Logger_Source_Detector::instance()->method()`
10. `composer dump-autoload`

**Note on soft singleton**: No external `new AcrossAI_Logger_Source_Detector()` callers exist (confirmed by audit), so `private function __construct()` is safe (unlike BerlinDB Table classes where `DEC-TABLE-SOFT-SINGLETON` applies).
