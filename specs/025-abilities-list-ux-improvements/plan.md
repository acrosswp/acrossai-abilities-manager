# Implementation Plan: Abilities List UX Improvements

**Branch**: `025-abilities-list-ux-improvements` | **Date**: 2026-06-02 | **Spec**: [spec.md](spec.md)

## Summary

Six targeted improvements to the Abilities Manager admin list view and Settings page: stateful pagination, an admin-controlled per-page setting, CSS-only hiding of the All/Published/Draft tabs, a "Clear All Overrides" row action, two new table columns (Description, Show in REST), and a localStorage-backed column visibility toggle. All changes are front-end-only except one PHP addition (`perPage` to the `acrossaiAbilitiesManager` inline script) and one PHP addition (new Settings API field). No REST schema changes needed — `has_override` already exists in every ability response.

---

## Technical Context

**Language/Version**: PHP 7.4+, JavaScript (ESNext/JSX), SCSS
**Primary Dependencies**: `@wordpress/element`, `@wordpress/data`, `@wordpress/i18n`, WordPress Settings API
**Storage**: `wp_options` (per-page setting), browser `localStorage` (column preferences)
**Testing**: PHPStan level 8, PHPUnit, ESLint
**Target Platform**: WordPress 6.9+ admin, desktop browsers
**Performance Goals**: No additional DB queries — `has_override` already computed in the existing merge; per-page value is a single `get_option()` call cached by WP's options cache
**Constraints**: PHP 7.4 union-type workaround via docblocks; must remain Plugin Check clean; must follow `AC-ENQUEUE-ADMIN`
**Scale/Scope**: ~100–500 abilities per site; localStorage key < 1 KB

### Key Implementation Corrections (vs. spec language)

- The spec refers to `window.acrossaiAbilities.perPage` — **the actual global object is `window.acrossaiAbilitiesManager`** (confirmed in `src/js/abilities/index.js:12`, `AbilityForm.jsx:39`, `api/client.js:11`). `perPage` must be added to `window.acrossaiAbilitiesManager`, not a new object.
- `has_override` is already computed by `AcrossAI_Ability_Merger` and returned by `AcrossAI_Abilities_Formatter` — no PHP change needed for that field.
- `.subsubsub` CSS selector already exists in `admin.scss` — hiding it requires only adding `display: none;` to the existing rule, not creating a new one.

## Technical Design

### T1 — Stateful Pagination (AbilitiesList.jsx)

**Current state**: `const [page] = useState(1)` — frozen. `const PER_PAGE = 20` — hard-coded.

**Change**:
```jsx
const [page, setPage] = useState(1);
const perPage = window.acrossaiAbilitiesManager?.perPage || 20;
const totalPages = Math.ceil(total / perPage) || 1;
```

Reset page on any filter/search change via a dedicated `useEffect`:
```jsx
useEffect(() => { setPage(1); }, [search, sourceFilter, statusFilter, sortDir]);
```

Pass `per_page: perPage` (not the old `PER_PAGE`) to `dispatch.fetchAbilities`.

**Pagination nav markup** — rendered inside `.tablenav` (above) and as a second `.tablenav-pages` div immediately after the `</table>` closing tag (below):
```jsx
<div className="tablenav-pages">
    <span className="displaying-num">{total} {__('items', 'acrossai-abilities-manager')}</span>
    <span className="pagination-links">
        <button className="button" disabled={1 === page} onClick={() => setPage(1)}>«</button>
        <button className="button" disabled={1 === page} onClick={() => setPage(p => p - 1)}>‹</button>
        <span className="paging-input">{page} {__('of', 'acrossai-abilities-manager')} {totalPages}</span>
        <button className="button" disabled={page >= totalPages} onClick={() => setPage(p => p + 1)}>›</button>
        <button className="button" disabled={page >= totalPages} onClick={() => setPage(totalPages)}>»</button>
    </span>
</div>
```

### T2 — Per-page Setting (SettingsMenu.php + admin/Main.php)

**SettingsMenu.php** — add to `register_settings()`:
```php
register_setting(
    'acrossai_abilities_settings',
    'acrossai_abilities_per_page',
    array(
        'sanitize_callback' => array( $this, 'sanitize_per_page' ),
        'default'           => 20,
    )
);

add_settings_section(
    'acrossai_display_settings_section',
    __( 'Display Settings', 'acrossai-abilities-manager' ),
    '__return_false',
    'acrossai-abilities-settings'
);

add_settings_field(
    'acrossai_abilities_per_page',
    __( 'Abilities per page', 'acrossai-abilities-manager' ),
    array( $this, 'render_per_page_field' ),
    'acrossai-abilities-settings',
    'acrossai_display_settings_section'
);
```

New public methods on `SettingsMenu`:
```php
public function sanitize_per_page( $value ): int {
    $int = absint( $value );
    return ( $int < 1 || $int > 200 ) ? 20 : $int;
}

public function render_per_page_field(): void {
    $value = (int) get_option( 'acrossai_abilities_per_page', 20 );
    printf(
        '<input type="number" id="acrossai_abilities_per_page" name="acrossai_abilities_per_page" value="%s" min="1" max="200" step="1" /><p class="description">%s</p>',
        esc_attr( (string) $value ),
        esc_html__( 'Number of abilities shown per page. Default: 20. Min: 1. Max: 200.', 'acrossai-abilities-manager' )
    );
}
```

**admin/Main.php** — extend the existing `wp_add_inline_script` array inside `is_manager_page()` block:
```php
'perPage' => (int) get_option( 'acrossai_abilities_per_page', 20 ),
```
This key is added to the existing `window.acrossaiAbilitiesManager` object — no new `wp_add_inline_script` call needed.

### T5 — Description and Show in REST Columns (AbilitiesList.jsx)

New cell renderers (follow `DEC-TYPECELL-REGISTRY-FALLBACK` — read item field first, fall back to `_registry`):
```jsx
function DescriptionCell({ item }) {
    const desc = item.description || item._registry?.description || '';
    if (!desc) return <span>—</span>;
    const short = desc.length > 80 ? desc.slice(0, 80) + '…' : desc;
    return <span title={desc}>{short}</span>;
}

function ShowInRestCell({ item }) {
    const val = item.show_in_rest ?? item._registry?.show_in_rest ?? false;
    return val
        ? <span className="mcp-y">{__('✓ Yes', 'acrossai-abilities-manager')}</span>
        : <span className="mcp-n">{__('○ No', 'acrossai-abilities-manager')}</span>;
}
```

Insert between Type and MCP columns in `<colgroup>`, `<thead>`, and every `<tbody>` row.

Update loading/empty-state `colSpan` — see T6 for dynamic approach.

### T6 — Column Visibility Toggle (AbilitiesList.jsx)

```jsx
const COLUMN_DEFAULTS = {
    label: true, category: true, source: true, status: true,
    type: true, description: true, show_in_rest: true, mcp: true,
};
const COLUMN_LABELS = {
    label: __('Label', '...'), category: __('Category', '...'),
    source: __('Source', '...'), status: __('Status', '...'),
    type: __('Type', '...'), description: __('Description', '...'),
    show_in_rest: __('Show in REST', '...'), mcp: __('MCP', '...'),
};
const LS_KEY = 'acrossai_abilities_columns';

function loadColumnPrefs() {
    try {
        return { ...COLUMN_DEFAULTS, ...JSON.parse(localStorage.getItem(LS_KEY) || '{}') };
    } catch { return { ...COLUMN_DEFAULTS }; }
}

const [visibleColumns, setVisibleColumns] = useState(loadColumnPrefs);
const [columnsOpen, setColumnsOpen] = useState(false);

function toggleColumn(key) {
    setVisibleColumns(prev => {
        const next = { ...prev, [key]: !prev[key] };
        try { localStorage.setItem(LS_KEY, JSON.stringify(next)); } catch {}
        return next;
    });
}

// Close panel on outside click
useEffect(() => {
    if (!columnsOpen) return;
    const handler = (e) => { if (!e.target.closest('.columns-toggle')) setColumnsOpen(false); };
    document.addEventListener('mousedown', handler);
    return () => document.removeEventListener('mousedown', handler);
}, [columnsOpen]);

// Dynamic colSpan
const visibleCount = Object.values(visibleColumns).filter(Boolean).length;
const tableColSpan = visibleCount + 3; // +1 checkbox, +1 Slug, +1 Actions
```

Columns button added to `.tablenav` row (before the `.tn-pages` div):
```jsx
<div className="columns-toggle" style={{ position: 'relative' }}>
    <button type="button" className="button" onClick={() => setColumnsOpen(o => !o)}>
        {__('Columns', 'acrossai-abilities-manager')} {columnsOpen ? '▴' : '▾'}
    </button>
    {columnsOpen && (
        <div className="columns-panel">
            {Object.keys(COLUMN_DEFAULTS).map(key => (
                <label key={key} className="columns-panel-item">
                    <input type="checkbox" checked={!!visibleColumns[key]} onChange={() => toggleColumn(key)} />
                    {' '}{COLUMN_LABELS[key]}
                </label>
            ))}
        </div>
    )}
</div>
```

Each hideable `<col>`, `<th>`, `<td>` wrapped: `{visibleColumns.label && <col className="col-lbl" />}` etc.

Loading/empty-state rows use `colSpan={tableColSpan}`.
