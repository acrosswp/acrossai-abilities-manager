# Internationalization (i18n)

## Text domain must match the plugin slug

The text domain must be identical to the plugin directory name (the slug used in the plugin header and on WordPress.org).

```
Plugin directory:  wp-content/plugins/my-awesome-plugin/
Text domain:       my-awesome-plugin
```

In the plugin header:
```php
/**
 * Plugin Name: My Awesome Plugin
 * Text Domain: my-awesome-plugin
 * Domain Path: /languages
 */
```

## Load the text domain on `init`

Load the text domain on the `init` hook, not `plugins_loaded`. `init` is the correct hook since WordPress 6.7+; it is also the approach that supports Just-in-Time (JIT) translation loading.

```php
// In includes/Main.php via the Loader:
$this->loader->add_action( 'init', $this, 'load_textdomain' );

public function load_textdomain(): void {
    load_plugin_textdomain(
        'my-awesome-plugin',
        false,
        dirname( plugin_basename( MYPLUGIN_FILE ) ) . '/languages/'
    );
}
```

## Translation functions

| Function | Use case | Returns |
|---|---|---|
| `__( 'text', 'domain' )` | Translate and return | string |
| `_e( 'text', 'domain' )` | Translate and echo | void |
| `_x( 'text', 'context', 'domain' )` | Translate with context (disambiguates identical strings) | string |
| `_ex( 'text', 'context', 'domain' )` | Translate with context and echo | void |
| `_n( 'singular', 'plural', $count, 'domain' )` | Singular/plural | string |
| `_nx( 'singular', 'plural', $count, 'context', 'domain' )` | Plural with context | string |

```php
// Simple return
$label = __( 'Save Settings', 'my-awesome-plugin' );

// Echo directly
_e( 'Save Settings', 'my-awesome-plugin' );

// Context — 'Post' as a noun vs 'Post' as a verb
$noun = _x( 'Post', 'noun: a blog post', 'my-awesome-plugin' );
$verb = _x( 'Post', 'verb: to publish', 'my-awesome-plugin' );

// Plural
printf(
    _n( '%d item found.', '%d items found.', $count, 'my-awesome-plugin' ),
    $count
);
```

## Always escape after translating

Never `echo __( ... )`. Use the combined escape+translate functions:

```php
// WRONG
echo __( 'Welcome back!', 'my-awesome-plugin' );
echo '<span title="' . __( 'Information', 'my-awesome-plugin' ) . '">';

// CORRECT — escapes HTML entities
esc_html_e( 'Welcome back!', 'my-awesome-plugin' );
echo esc_html__( 'Welcome back!', 'my-awesome-plugin' );

// CORRECT — for attribute values
echo '<span title="' . esc_attr__( 'Information', 'my-awesome-plugin' ) . '">';

// CORRECT — in a printf / sprintf context
printf(
    '<p>%s</p>',
    esc_html__( 'Settings saved.', 'my-awesome-plugin' )
);
```

| Escape+translate function | Context |
|---|---|
| `esc_html__()` | HTML text content, return |
| `esc_html_e()` | HTML text content, echo |
| `esc_attr__()` | HTML attribute value, return |
| `esc_attr_e()` | HTML attribute value, echo |

## JavaScript i18n

1. Register translations for your script handle using `wp_set_script_translations()`:

```php
public function enqueue_scripts(): void {
    wp_enqueue_script(
        'my-awesome-plugin-app',
        MYPLUGIN_URL . 'build/js/app.js',
        [ 'wp-i18n' ],
        MYPLUGIN_VERSION,
        true
    );

    // Must be called after wp_enqueue_script
    wp_set_script_translations(
        'my-awesome-plugin-app',    // Script handle
        'my-awesome-plugin',        // Text domain
        MYPLUGIN_PATH . 'languages' // Path to .json translation files
    );
}
```

2. Use `@wordpress/i18n` in your JavaScript/TypeScript source:

```js
import { __, _n, _x, sprintf } from '@wordpress/i18n';

const label   = __( 'Save Settings', 'my-awesome-plugin' );
const message = sprintf(
    _n( '%d item', '%d items', count, 'my-awesome-plugin' ),
    count
);
```

3. The JSON translation file is named `{domain}-{locale}-{handle-md5}.json` and lives in `languages/`. `wp i18n make-json` generates these from `.po` files.

## Generating and maintaining translation files

```bash
# 1. Generate the .pot (Portable Object Template) from PHP and JS source:
wp i18n make-pot . languages/my-awesome-plugin.pot \
    --domain=my-awesome-plugin \
    --exclude=vendor,node_modules,build

# 2. Create a .po file for a locale (copy the .pot, rename it):
cp languages/my-awesome-plugin.pot languages/my-awesome-plugin-de_DE.po

# 3. Translate strings in Poedit (GUI) or Loco Translate (WP plugin).

# 4. Compile .po → .mo (required for PHP translations):
msgfmt -o languages/my-awesome-plugin-de_DE.mo languages/my-awesome-plugin-de_DE.po

# 5. Generate JSON files for JavaScript translations:
wp i18n make-json languages/ --no-purge
```

Regenerate the `.pot` file after every release cycle. Commit `.pot`, `.po`, `.mo`, and `.json` files to version control.

## Checklist before release

- [ ] Every string visible to users is wrapped in a translation function
- [ ] No bare `echo __()` — always use `esc_html_e()` / `esc_attr_e()`
- [ ] Text domain in plugin header matches directory name
- [ ] `load_plugin_textdomain()` is called on `init`
- [ ] `wp_set_script_translations()` called for every handle that uses `@wordpress/i18n`
- [ ] `.pot` file updated and committed
- [ ] `languages/` directory included in the plugin archive
